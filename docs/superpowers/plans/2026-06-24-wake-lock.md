# Wake Lock — Plan de implementación

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Mantener la pantalla encendida mientras hay una canción abierta, usando la Screen Wake Lock API.

**Architecture:** Variable de módulo `wakeLock` con el sentinel. Se adquiere en `abrirDetalle`, se libera en `cerrarDetalle`, y se re-adquiere en `visibilitychange` si la app vuelve a primer plano con una canción abierta. Feature-detection: sin soporte, no-op.

**Tech Stack:** JS vanilla en `index.html`. Verificación con Claude_Preview (`preview_eval` + console logs).

## Global Constraints

- Todo en `index.html`; sin dependencias; sin UI nueva.
- Feature-detection de `navigator.wakeLock`; sin soporte la app funciona igual.
- Errores de la API atrapados y silenciados (es una mejora, no crítica).

- [ ] **Step 2: Agregar las funciones de wake lock** — después de la declaración de globales (antes de `parseChordPro`)

```js
  // ─── Wake Lock (pantalla activa al leer una canción) ─────
  async function solicitarWakeLock() {
    if (!('wakeLock' in navigator)) return;
    try {
      wakeLock = await navigator.wakeLock.request('screen');
    } catch (e) { /* batería baja / denegado: ignorar */ }
  }

  async function liberarWakeLock() {
    if (wakeLock) {
      try { await wakeLock.release(); } catch (e) {}
      wakeLock = null;
    }
  }

  document.addEventListener('visibilitychange', () => {
    if (document.visibilityState === 'visible' && cancionActual && !wakeLock) {
      solicitarWakeLock();
    }
  });
```
